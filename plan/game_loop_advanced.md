# Game Design Summary — Core Loop & Systems (Revised)

## 1. Core Concept
Game is built around a **three-layer progression system**:

- Exploration (World)
- Transformation (Camp)
- Adaptation (Mechanae + Enemy Response)

The game focuses on:
> continuously changing how the player fights and survives, not raw stat growth.

---

## 2. Core Resources

### Structurix
- Foundation material
- Defines durability / structural integrity of gear
- Required in all equipment

### Secturix
- Core combat material
- Defines weapon type (damage + speed profile)
- Main progression resource for combat effectiveness

### Mechanae
- Single permanent modifier per item
- Changes weapon behavior, not raw scaling

Examples:
- area effects / crowd interaction
- status interaction changes (ignite, poison, etc.)
- conditional bonuses (low HP, surrounded, etc.)
- defensive or utility transformations

Rule:
> Mechanae modifies rules, not numbers.

---

## 3. Combat System

Weapons are composed of:

- Structurix → durability / stability
- Secturix → combat identity (DPS archetype)
- Mechanae → rule modifier (behavior change)

Key rule:
> No perfect build exists. Every choice has trade-offs.

---

## 4. Enemy System

Enemies adapt based on world progression:

After boss progression:
- behavior changes (not only stats)
- tactical patterns evolve:
  - swarm behavior
  - hunting behavior
  - flanking / surrounding behavior

Modifiers rotate rather than stack:
- damage → speed → durability → coordination

---

## 5. World Structure

### Zones
- Camp (safe transformation hub)
- Danger zones (resource + pressure zones)
- Boss zones (progress gates)

World is:
> infinite cave system with reactive behavior layers

---

## 6. Camp System

### Core Role
Camp is NOT survival infrastructure.

Camp is:
> the only place where the player can transform their build and reset their identity

---

### Camp Expansion (Physical Growth)

Camp expands physically over time.

Expansion:
- increases available working space
- allows more simultaneous crafting/processing setups
- visually reflects progression

Rule:
> expansion improves capacity and scale of operations, not new systems

---

## 7. Crafting Philosophy

- Full crafting and modification only in camp
- Field crafting is limited to:
  - survival actions
  - basic repairs
  - minimal maintenance

Camp = transformation
World = execution

---

## 8. Structure System (World Interaction)

Player-built structures outside camp:

- influence enemy behavior patterns
- increase or decrease local pressure
- act as environmental anchors

Structures do NOT form an economy.
They create:
> shifts in world difficulty and behavior

---

## 9. Core Gameplay Loop

1. Spawn in camp
2. Choose Mechanae (define playstyle)
3. Enter world (expedition)
4. Fight and gather Structurix / Secturix
5. Build structures → influence enemy behavior
6. Face increasing tactical pressure
7. Reach boss gate (mechanical challenge)
8. Defeat boss → world behavior changes
9. Return to camp
10. Rebuild / adjust loadout
11. Repeat loop

---

## 10. Design Principles

- No permanent stat inflation
- Progression = behavior change, not numbers
- One Mechanae modifier per item
- Camp = transformation hub
- World = reactive system

---

## 11. Core Identity

> The player does not become stronger in a linear way.
> The player continuously changes the rules of how strength works.
